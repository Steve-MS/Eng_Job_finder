## What was delivered

### 7 Acceptance Dimensions

**1. Per-Source Adapter Acceptance (5 gates)**
- Data availability: ≥10 listings per adapter per run, or explicit "none found" flag
- Error handling: All exceptions logged, zero crashes, graceful retry/backoff (exponential, max 3 retries)
- robots.txt compliance: Adapter respects Crawl-delay and /disallow rules; logs compliance check result
- HTML schema change detection: Adapter logs warning if expected CSS selectors fail on ≥20% of listing page samples (indicates site redesign; manual review required)
- Rate limiting: Adapter enforces configured delays (2–3 s HTML, 10 s Jobiqo); never triggers 429/403

**2. Extraction Accuracy on Gold Set (50+ hand-labelled listings)**
- Per-field precision/recall:
  - **title:** precision ≥88%, recall ≥90% (name of role must be unambiguous)
  - **employer/company:** precision ≥90%, recall ≥92%
  - **day_rate (min/max):** precision ≥80%, recall ≥85% (rates can be ranges or single values; both acceptable)
  - **ir35_status:** precision ≥70%, recall ≥60% (hardest field; "unknown" is acceptable fallback)
  - **location:** precision ≥95%, recall ≥98% (must normalize to UK region)
  - **contract_type:** precision ≥98%, recall ≥95% (binary: contract/non-contract)
  - **duration_weeks:** precision ≥75%, recall ≥70% (can be range or open-ended; both OK)

**3. Filter Precision/Recall (100+ gold-set positives + negatives)**
- **contract_type filter:** precision ≥98% (reject permanents correctly), recall ≥92% (accept all true contracts)
- **UK geographic filter:** precision ≥99%, recall ≥95% (zero non-UK leakage)
- **PM role filter:** precision ≥92%, recall ≥88% (accept Project/Programme/Product/Portfolio variants; reject PMC, PA, Coordinator)
- **mechanical domain filter:** precision ≥96%, recall ≥90% (accept mechanical, M&E, MEP, rail, aerospace, energy, manufacturing, automotive, plant, equipment; reject generic "engineering", "civil", "software")

**4. Dedup Quality on Hand-Labelled Pairs (100+ pairs)**
- **Precision ≥98%:** Zero false merges (different listings merged as same)
- **Recall ≥88%:** Catch ≥88% of true duplicate pairs (same listing from different sources)
- Jaro-Winkler >0.85 on title + employer/agency matching + 14-day posting proximity heuristic

**5. E2E Pipeline Acceptance (end-to-end execution)**
- **Execution time ≤15 minutes:** All 7 adapters (Reed + 6 scrapers) complete within 15 min from orchestrator start
- **Non-empty output or explicit "none found":** Pipeline always produces a result status (never silent failure)
- **Graceful partial failure:** If 1 adapter crashes, remaining 6 continue; crashed adapter logged; final report includes coverage note (e.g., "⚠️ CWJobs adapter failed; coverage: Reed, Totaljobs, RailwayPeople, Energy Jobline, The Engineer Jobs, Aviation Job Search")
- **Data persistence for replay:** Every run persists raw and normalized listings to SQLite with run_id; audit trail available
- **100% schema validation:** Every normalized listing passes 28-field schema check (no null required fields except those explicitly optional)

**6. Report Quality Acceptance**
- **Hyperlinks ≥99% valid:** All source_urls must 200-OK or 301 (no 404/403 leakage)
- **Zero duplicates in final output:** Dedup merged all duplicates; final report contains no duplicate listing_ids
- **Outlier flagging 100%:** All sanity-check triggers (low rate, high rate, past start, vague location, etc.) rendered as ⚠️ flag in report
- **Clean rendering:** Markdown renders without parse errors; flags display correctly; no embedded HTML/SQL injection
- **Metadata completeness:** Every report includes source coverage, run timestamp, dedup stats, row count

**7. Out-of-Scope Clarifications for v0.1 (explicit deferred)**
- Freshness SLA <24h: Not enforced v0.1 (manual trigger OK; 4-hourly optional v0.2)
- Inside/outside IR35 inference: Not inferred; only extracted/normalized if present in listing
- Cross-year dedup: Only 14-day heuristic v0.1; cross-year historical tracking deferred v0.2
- Company name normalization: No standardization v0.1 (e.g., "ABC Ltd" vs "ABC Limited" kept as separate companies)
- NLP PM specialization inference: No sub-role distinction (e.g., "Delivery PM" vs "Portfolio PM" not categorized); title regex only
- Browser automation (Playwright): Deferred v0.2 if JS-heavy sites block scraping

## Why this matters
Acceptance criteria prevent scope creep and define MVP vs v0.2 boundary. **Precision-first asymmetry:** false positives (junk listings in report) worse than false negatives (missed real jobs); hence higher precision targets (98–99%) than recall (88–92%). Concrete thresholds (Jaro-Winkler >0.85, ≤15 min exec, ≥10 listings/adapter) are measurable and testable.
