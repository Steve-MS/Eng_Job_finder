## Key Components Implemented
1. **28-field NormalizedListing schema:** listing_id, source, source_url, title, employer, agency, location, location_region, day_rate_min, day_rate_max, duration_weeks, start_date, end_date (inferred), contract_type, remote_policy, ir35_status, description_clean, sector, job_level, keywords, last_updated, discovered_at, last_seen_at, confidence_score, extraction_method, raw_listing_id, created_at, updated_at
2. **3-tier extraction:** structured (API direct) → regex fallback (title, rate, date, location) → LLM fallback (free text description mining)
3. **Sector enum:** rail, aerospace, defence, energy, nuclear, construction, maritime, automotive, process, generalist (dual population: source default + keyword map)
4. **Filters:** contract_type mandatory, UK-only mandatory, PM-role title filter, mechanical engineering domain filter (with cross-sector awareness for verticals)
5. **Dedup:** Jaro-Winkler >0.85 on title + employer/agency + location, rate band overlap check, 14-day posting proximity
6. **Storage:** SQLite two-table design (raw_listings for audit trail, normalized_listings for reporting)
7. **Dependencies:** rapidfuzz >=3.0 (hard production dep), openai >=1.0 (soft dep, only if MECHPM_LLM_FALLBACK=1)
